# 第2章 编程示例说明

本目录包含《数字智能化设计：原理、应用与未来》第2章"数学与算法基础"中 **2.5 编程示例** 的三个 Python 实现，围绕**深空探测**场景验证线性代数、概率论与算法设计的工程可行性。

---

## 环境要求

- Python 3.10+
- NumPy（`pip install numpy`）
- pgmpy ≥ 1.1.0（`pip install pgmpy`）

---

## 文件说明

### 2_5_1.py — 基于矩阵运算的探测器姿态控制系统模拟（对应 2.5.1 节）

**理论背景**：探测器姿态的本质是空间中刚体坐标系的旋转，采用 3×3 方向余弦矩阵（DCM）描述，对应第 2.1 节的矩阵运算理论。

**功能**：
1. 构建初始姿态矩阵（单位矩阵，表示无旋转）
2. 定义绕 X/Y/Z 主轴的旋转操作（标准右手定则，角度单位为弧度）
3. 模拟复合姿态调整：先绕 Z 轴偏航 45°，再绕 Y 轴俯仰 30°，最后绕 X 轴横滚 60°
4. 验证正交性（$R^T R \approx I$）并输出机体坐标轴方向

**运行方式**：
```bash
python 2_5_1.py
```

**预期结果**：
- 更新后的姿态矩阵为合法旋转矩阵
- 正交性验证：对角线元素 ≈ 1，非对角线元素 ≈ 1e-17（浮点误差范围内）
- 三个机体轴方向向量两两正交、模长均为 1

**核心注意**：旋转矩阵需右乘当前姿态矩阵（`attitude @ Rz @ Ry @ Rx`），表示相对于当前机体坐标系的旋转；左乘表示相对于惯性坐标系，不符合机体控制逻辑。

### 2_5_2.py — 基于贝叶斯定理的探测器故障诊断（对应 2.5.2 节）

**理论背景**：利用贝叶斯定理（第 2.2.2 节）构建概率图模型，根据传感器观测推断各子系统的故障后验概率。

**探测器模型**：

```
PowerFault ──→ ThermalFault
     │                │
     └────────┬────────┘
              ↓
        AttitudeFault
```

**功能**：
1. 构建包含电源（Power）、热控（Thermal）、姿态（Attitude）三个子系统的贝叶斯网络
2. 设定先验概率（Power 故障率 2%）与条件概率表（CPD）
3. 模拟传感器观测：姿态控制系统出现异常（`AttitudeFault = 1`）
4. 计算电源和热控系统的故障后验概率

**运行方式**：
```bash
python 2_5_2.py
```

**预期结果**：

| 子系统 | 先验故障率 | 观测姿态故障后的后验故障率 |
|-------|-----------|------------------------|
| 电源（PowerFault） | 2% | ≈ 22% |
| 热控（ThermalFault）| ~1% | ≈ 13% |

后验概率大幅高于先验，体现了贝叶斯"上游原因被激活"的推理机制。

**依赖说明**：pgmpy ≥ 1.1.0 已将 `BayesianNetwork` 重命名为 `DiscreteBayesianNetwork`，代码已通过别名兼容处理，无需降级库版本。

### 2_5_3.py — 分治算法在探测器数据处理中的应用（对应 2.5.3 节）

**理论背景**：利用归并排序（Merge Sort，第 2.4.2.1 节）对大规模天体观测数据进行高效处理，时间复杂度 $O(n \log n)$，空间复杂度 $O(n)$。

**功能**：
1. 模拟生成 10000 个天体亮度数据（均匀分布，范围 0~1000）
2. 使用归并排序对亮度数据排序
3. 提取亮度最强的前 10 个星体
4. 统计亮度高于阈值（900）的星体数量
5. **扩展**：对包含亮度与视速度的结构化多维数据按指定键排序

**运行方式**：
```bash
python 2_5_3.py
```

**预期结果**：
- 最强前 10 个星体亮度均接近 1000
- 亮度 > 900 的星体约占总数的 10%（约 1000 个）
- 结构化排序输出含亮度和视速度的字典列表，按亮度升序排列
